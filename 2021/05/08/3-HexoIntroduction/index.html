    1. 介绍

    3-hexo 是为 hexo 制作的主题。三段式设计、方便。

    之所以选择这个主题，有以下原因：

    1. 布局简洁明了
    2. 当前版本已经方便地解决了数学公式显示问题

    接下来，将从如何安装该主题开始，详述基于该主题的Hexo的使用方法，其他主题也是大同小异。

    2. 3-Hexo主题初始化

    2.1. 安装

    在你的博客目录下，打开 Git，执行如下指令，克隆该主题到本地：

    git clone https://github.com/yelog/hexo-theme-3-hexo.git themes/3-hexo

    用 VS Code打开 .\_config.yml

    找到如下的代码，修改为如下形式：

    theme: 3-hexo

    2.2. 更新

    在你的博客目录下，打开 Git，执行如下指令，更新主题：

    cd themes/3-hexo
git pull

    3. 3-Hexo 主题设置

    3.1. 设置头像与图标

    用 VS Code打开 ./themes/3-hexo/_config.yml

    找到如下的代码，将自己的头像图片放到 ./themes/3-hexo/source/img 里：

    # 你的头像url
avatar: /img/avatar.png
favicon: /img/avatar.png

    3.1. 语言国际化

    用 VS Code打开 .\_config.yml

    找到如下的代码，修改为如下形式：

    language: zh-CN

    3.3. 关闭 Hexo 自带代码高亮

    用 VS Code打开 .\_config.yml

    找到如下的代码，修改为如下形式：

    highlight:
  enable: false

    3.4. 数学公式支持

    用 VS Code打开 ./themes/3-hexo/_config.yml

    找到如下的代码，修改为如下形式：

    # MathJax 数学公式支持
mathjax:
  on: true #是否启用
  per_page: false # 若只渲染单个页面，此选项设为false，页面内加入 mathjax: true

    数学公式样例： \[ f(x)=e^x+\pi \]

    由于 Hexo 自带的 math 渲染器并不好用，这里给出更完善的解决方案。

    首先安装 Pandoc，在这里下载Win10版本的pandoc。

    在博客目录下，执行如下命令来更换渲染器：

    npm uninstall hexo-renderer-marked --save
npm install hexo-renderer-pandoc --save

    渲染后如果如下公式换行显示正常，则成功！

    a^2\\
b_2

    \[ a^2\\ b_2 \]

    3.5. 字数统计

    先在博客根目录执行指令安装字数统计插件：

    npm i hexo-wordcount --save

    然后用 VS Code打开 ./themes/3-hexo/_config.yml

    找到如下的代码，修改为如下形式：

    word_count: true
# true 开启字数统计
# 开启此功能需要安装插件 ：在 hexo根目录 执行npm i hexo-wordcount --save

    3.7. 开启关于页面

    在博客根目录执行如下指令：

    hexo new page "about"

    该页面的文件将生成为 ./source/aoubt/index.md ，根据你的需要进行编辑。

    用 VS Code打开 ./themes/3-hexo/_config.yml

    找到如下的代码，修改为如下形式，打开关于设置：

    # 左下角自定义菜单
menu:
  about:  # '关于' 按钮
    on: true # 是否显示
    url: /about  # 跳转链接
    type: 1 # 跳转类型 1：站内异步跳转 2：当前页面跳转 3：打开新的tab页

    3.8. 代码高亮

    请先完成 3.3.关闭 Hexo 自带代码高亮。

    用 VS Code打开 ./themes/3-hexo/_config.yml

    找到如下的代码，修改为如下形式：

    highlight:
  on: true # true开启代码高亮，开启需要关闭博客 _config.yml 中的 highlight
  lineNum: false # true显示行号
  copy: true # 复制功能
  theme: atom-dark
# 代码高亮主题,效果可以查看 https://highlightjs.org/static/demo/
# 支持主题：

# darcula : 参考idea中的darcula的主题

# atom-light : 参考Atom的light主题
# github : 参考GitHub版的高亮主题
# github-gist : GitHub-Gist主题
# brown-paper : 牛皮纸效果
# gruvbox-light : gruvbox的light主题
# gruvbox-dark ： gruvbox的dark主题
# rainbow :
# railscasts :
# sunburst :
# kimbie-dark :
# kimbie-light :
# school-book : 纸张效果
# zenbum : oschina 也选择此高亮主题

    小明 98 76

    找到如下的代码，修改为如下形式：

    3.10. 图片管理

    安装图片管理插件：

    npm install hexo-asset-image --save

    用 VS Code打开 .\_config.yml

    找到如下的代码，修改为如下形式：

    post_asset_folder: true

    当资源文件管理功能打开后，Hexo将会在你每一次通过 hexo new [layout] <title> 命令创建新文章时自动创建一个文件夹。这个资源文件夹将会有与这个文章文件一样的名字。将所有与你的文章有关的资源放在这个关联文件夹中之后，你可以通过相对路径来引用它们，这样你就得到了一个更简单而且方便得多的工作流。

    4. 文章编辑操作

    4.1. 文章创建

    创建新文章时，总得有个模板，Hexo 的文章模板保存在目录 ./scaffolds 下。

    你可以执行下列命令来创建一篇新文章。

    hexo new [layout] <title>

    [layout] 是可选项，若缺省，则默认为 post。

    我的 post 模板为（./scaffolds/post.md）：

    ---
title: {{ title }}
date: {{ date }}
categories:
tags:
---

    比如我创建一篇以 post 为模板，名为 test 的文章，使用如下指令：

    hexo new post "test"

    如下指令与上面的指令等价：

    hexo new "test"

    指令执行后，在目录 ./source/_posts 下会生成对应的 test.md 文件。

    直接打开编辑你的文章就行了。

    4.2. 文章删除

    直接在目录 ./source/_posts 下删除对应的文件，然后执行缓存清理指令即可：

    hexo clean

    4.3. 文章分类

    ---
title: {{ title }}
date: {{ date }}
categories:
- A
- B
— C
tags:
- tag1
- tag2
---

    在文章的头部对应部分，可以直接编辑分类信息。

    比如上面的示例中，categories 下面，A 是 B 的父分类，B 是 C 的父分类，可以一直循环下去。

    tags 是对应的标签。

    分类信息将在左侧边栏显示，标签可用于搜索栏中使用 #tag1 检索文章。

    4.4. 文章置顶

    ---
title: {{ title }}
date: {{ date }}
top: x
categories:
- A
- B
— C
tags:
- tag1
- tag2
---

    在文章文件头部加上 top:x ，主题将自动为其置顶，x 为置顶权重，若有多个文章设置了置顶，将按照置顶权重排序。
