    本文将详细介绍如何利用 Hexo 与 Github Pages 构建一个个人主页。

    本文的配置环境为：

    版本 Windows 10 教育版 版本号 20H2 操作系统内部版本 19042.964

    接下来将按照如下步骤来讲解构建过程：

    1. 搭建本地 Hexo 博客
    2. 在Github建立Page仓库，并与本地Git建立SSH
    3. 将本地 Hexo 博客部署到 Github Pages 上去

    值得一提的是，步骤1 与 步骤2 的完成不存在先后顺序，步骤3 必须在前两个步骤完成后再进行。

    1. 介绍

    • Hexo

      Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

    • Github Pages

      Gihub Pages 是Github官方提供给用户的个人主页，结合 Hexo 生成静态网页的特点，可以用来搭建博客。

    2. 准备工作

    2.1. 安装 Git

    Git 是一个分布式版本控制软件，最初由林纳斯·托瓦兹创作，于2005年以GPL发布。最初目的是为更好地管理Linux内核开发而设计。

    Git 的安装是建立博客过程中最关键的部分，该软件要承担 终端命令 和 部署到Github 的重任，不过别担心，Windows上的安装非常简单，在这里下载，默认安装就好。

    2.2. 安装 node.js

    Node.js 是能够在服务器端运行 JavaScript 的开放源代码、跨平台运行环境。

    Node.js 是 Hexo 运行的基础，在这里下载，下载完默认安装就好，非常简单。

    下载时请选择长期支持版（LTS）！

    2.3. 安装 Typora

    Typora 是一个非常好用的 Markdown 编辑器，给你实时渲染的体验，软件支持全平台。

    在编写 Markdown 博文时，推荐使用该软件进行编辑。

    Windows 版本在这里下载。

    2.4. 安装 Vs Code

    Visual Studio Code（简称VS Code）是一款由微软开发且跨平台的免费源代码编辑器。

    都用Github建博客了，这个干嘛用的肯定知道！肯定装了！我就不多说了！（官网）

    3. 搭建本地博客

    这个部分其实建议参考 Hexo 的官方文档，我这里仅给出我的步骤。

    注意：请先完成准备工作！

    3.1. 安装 Hexo

    随便在哪里右键鼠标，点击 Git Bash Here，执行如下命令：

    npm install -g hexo-cli

    安装完毕后，执行 hexo -v，若出现类似如下内容，则说明安装成功：

    yzn@DESKTOP MINGW64 ~/Desktop

    3.2. 初始化博客模板

    在你想要保存博客文件的目录下，右键鼠标，点击 Git Bash Here，

    执行初始化命令：

    hexo init myblog

    执行完毕后，在当前目录下，就会自动创建一个名为 myblog 的目录了。

    进入目录后，执行服务器启动指令，就能在 http://localhost:4000/看见创建好的博客模板了。

    cd myblog   # 进入博客目录
hexo server # Hexo本地服务器启动

    4. Github Pages

    4.1. 创建 Github Pages 仓库

    在个人的 Github 中建立一个仓库，仓库名设置为 xxx.github.io，xxx 是你Github的用户名。

    4.2. 配置 SSH Key

    这一步是为了部署本地博客到 Github 上去。

    如果你的 Git 是刚安装好的，请执行下面的指令：

    # 全局配置本地账户
git config --global user.name "xxx"
git config --global user.email "邮箱地址"
# 生成密钥 SSH Key
ssh-keygen -t rsa -C '上面的邮箱'
# 按三次回车即可生成SSH Key

    生成完毕后，执行如下指令查看 SSH Key

    cat ~/.ssh/id_rsa.pub

    打开你的 Github 主页，点击右上角你的头像，再在出现的菜单中点击 Settings，进入 SSH设置页面，在 SSH Keys 部分，点击按钮 New SSH key，将刚刚执行指令获得的以 "ssh-ras" 开头的所有字符串复制到 Key 的部分去，Title 随便设置，看你心情。

    上面的部分完成后，执行指令 ssh -T git@github.com 检查是否配置成功，若出现形如如下情况，则成功：

    yzn@DESKTOP MINGW64 ~/Desktop
$ ssh -T git@github.com
Hi yznnnn! You've successfully authenticated, but GitHub does not provide shell access.

    5. 部署本地Hexo博客到Github Pages

    先安装一个插件，用于部署本地Hexo博客到Github Pages：

    npm install hexo-deployer-git --save

    安装完毕后，首先设置部署地址，用 VS Code打开 .\_config.yml

    找到如下的代码，修改为如下形式：

    # Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
   type: git
   repo: git@github.com:yznnnn/yznnnn.github.io.git
   branch: master

    在本地生成静态页面：

    hexo clean     # 清除缓存 
hexo generate  # 生成静态页面

    然后部署到 Github Pages 中去：

    hexo deploy

    如果一切顺利，就能在 https://xxx.github.io/ 上看到你的页面啦！

    如果显示404，请先去上个厕所回来再看！

    部署命令可以集成为如下一条命令（根据这个命令，你可以DIY一下自己的部署命令）：

    hexo clean && hexo generate && hexo deploy && hexo server
